# Система телеметрії
Практична частина курсової роботи 
Романчук Анни КН-3

Проєкт являє собою спрощений прототип системи телеметрії для моніторингу системних ресурсів комп’ютера.


## Встановлення бібліотеки

```bash
python --version
pip install psutil
```
## Запуск системи

```
python main.py
```

## Що реалізовано

Система виконує такі функції:

- збір системних метрик з локального хоста;
- контроль порогових значень;
- збереження метрик у вигляді часових рядів;
- надання даних через HTTP API;
- відображення метрик у web-dashboard.

## Основні метрики

У проєкті відстежуються такі показники:

- завантаження процесора (CPU);
- використання оперативної пам’яті (RAM);
- використання дискового простору (Disk);
- мережевий трафік (Network In / Network Out);
- кількість процесів у системі.

## Структура проєкту

- `main.py` — головний модуль запуску системи;
- `agent.py` — збір системних метрик;
- `alerts.py` — перевірка порогів та формування сповіщень;
- `data_base.py` — in-memory база даних часових рядів;
- `server.py` — HTTP-сервер та API;
- `dashboard.html` — web-інтерфейс для візуалізації.

## Вимоги

Для запуску потрібен:

- Python 3
- бібліотека `psutil`
